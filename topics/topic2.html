Variables & Data Types

Variables are names that refer to values in memory. In Python variables are created when first assigned — there's no explicit declaration. Python is dynamically typed, meaning a variable can hold any type and its type can change at runtime.

Built-in data types

  • Numbers: int (arbitrary precision), float (double-precision), complex.
  • String: immutable sequence of Unicode characters.
  • Boolean: True / False.
  • Sequence types: list (mutable), tuple (immutable), range.
  • Mapping: dict (key → value store).
  • Set: unordered collection of unique items.
  • None: special null-like singleton used for “no value”.

Examples and idioms

# variables and types
x = 10          # int
pi = 3.14159    # float
name = "Vipul"  # str
flags = [True, False]
user = {"id":1, "name":"Vipul"}

Use tuple packing/unpacking for multiple returns: 

def divmod_pair(a, b):
    return a // b, a % b

q, r = divmod_pair(10, 3)

Type conversion

Convert explicitly with constructors: int(), float(), str(), list(), dict(). Rely on duck-typing: functions accept objects that implement expected behavior rather than specific types.

Best practices

  • Name variables clearly (snake_case)
  • Prefer immutable types where possible (tuples) to avoid accidental mutation
  • Use type hints (PEP 484) for larger projects: def add(a: int, b: int) -> int:

Summary

Understanding Python’s core types is essential. Practice by manipulating lists, dictionaries, and strings; learn common methods (append, pop, split, join) and when to use built-in functions (map, filter, sorted).
